Индекс к файлу test_const_rules_samples.txt
(какой фрагмент к какой проверке относится; ожидаемый слой: лексика / синтаксис / семантика)

Успешные
  T_OK_* — без ошибок на всех этапах.

Лексика
  T_LEX_AT, T_LEX_HASH, T_LEX_PLUS — недопустимый символ (не из алфавита лексера).
  T_LEX_FLOAT — дробный литерал; сообщение парсера про целое число.

Синтаксис (парсер), по блокам в samples-файле
  Только ;;;; — пропущено ключевое слово const (одна точка с запятой / несколько).
  cont T_TYPO — ожидается const, найдено другое слово (опечатка con…).
  i32 T_NO_CONST — в начале ожидается const, а не тип данных.
  const i32: i32 — тип данных вместо идентификатора имени константы.
  i32 i32 = 1 — пропущен const в начале (паттерн «тип тип =»).
  100 T_NUM_BEFORE — пропущен const; число вместо имени.
  const ; T_STRAY_SEMI — лишняя ; перед именем.
  const ;; T_MULTI_SEMI — несколько лишних ; перед именем.
  ; T_SPURIOUS — лишний токен в начале (не const/имя/тип/число перед «имя :»).
  const const T_DUP_CONST — повтор const вместо имени.
  const T_EXTRA A — лишний идентификатор перед «:».
  const T_NO_COLON i32 — пропущен «:» между именем и типом.
  const T_EQ_BEFORE_TYPE: = i32 — «=» перед типом; часто ещё и пропущен «=» у значения.
  const T_NO_ASSIGN: i32 1 — пропущен «=».
  const T_NO_NUM: i32 = — нет числового литерала после «=».
  const T_SEMI_BEFORE_EQ — лишняя ; перед «=».
  const T_SEMI_BEFORE_NUM — лишняя ; перед числом.
  const T_NO_FINAL_SEMI — нет завершающего «;» (конец файла/строки).
  T_DOUBLE_COLON:: — повторяющийся «:» (2 раза).
  T_DOUBLE_EQ: i32 == — повторяющийся «=».
  T_DOUBLE_SEMI: … 1;; — повторяющийся «;».
  ??? garbage — неожиданный токен или неожиданная последовательность и синхронизация.

Семантика (semantic_analysis.py)
  T_DUP_A дважды — повторное объявление идентификатора.
  T_RANGE_U8 = 300 — значение вне диапазона u8.
  T_RANGE_I8 = 200 — значение вне диапазона i8.
  T_USE_BEFORE = T_UNDEF_REF при объявлении T_USE_BEFORE раньше T_UNDEF_REF — использование без предшествующего объявления.

Замечание
  Ветка «неизвестный тип для проверки диапазона» в семантике сейчас не срабатывает на стандартных типах лексера (все они есть в TYPE_RANGE). Она сработает, если позже добавить новый токен TYPE в лексер, но не добавить диапазон в TYPE_RANGE.
